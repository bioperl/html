bio.perl.org user info
User accounts
We have no clear policy right now on issuing user accounts to people. I would guess that users would fall into the following categories:
  • Active bioperl core module developers
  • Bioperl members who expect to be adding bug fixes or new code to the source tree via the CVS repository
  • Bioperl or "unofficial" members who have standalone bio-related perl code that they wish to place under CVS or on our anon-FTP server

I will be attempting to create an "anonymous" way for people to checkout the source code via CVS. This system will allow anyone to access our code but will not allow checkin, merge or commit actions.

People requesting accounts should email dag@sonsorol.org

Working with CVS

To use the CVS system remotely you must have

  • An account on bio.perl.org
  • secure shell (ssh) on your local machine
  • cvs on your local machine
  • a local environment variable 'CVS_RSH' set to ssh

System integrity disclaimer :)
Our system is running Linux on a donated i486 system. Methods for mirroring and backup of data will be haphazzard at best for a while. Please do not leave critical or unique data on this machine...keep copies of all your code locally. You have been warned.

There is a chance early next year that we can move onto some new pentium or SGI hardware. This will have mirrored disks and a dedicated backup device.

Network integrity disclaimer :)
bio.perl.org is sitting on my home network which is connected to the net via ISDN. Bandwith is dynamic from 56-112k according to demand. Electrical problems and the whims of the telephone company both contribute to my network dropping off the internet every once in a while. Sometimes these problems fix themselves and sometimes things have to wait until I get home from work :)

I am slowly working through the process of getting approval to host this machine at my company. Sticking a machine on the corporate network yet having it isolated outside the firewall is proving a hard concept to explain to management types. More info when I have it.
