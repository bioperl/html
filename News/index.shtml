Project News

[July / August / September 2001]

  • The Bioperl 0.9.0 developer release is available. Developer releases are not suitable for production use and are intended primarily for people interested in seeing new modules and functionality before they appear in official releases.

  • Jason Stajich has created a scripts repository and is seeking submissions. The goal is to have more utility and example scripts in our upcoming releases.

  • Nathan Siemers has released a perl-based gene expression package called Xobjects.

  • The Open Bioinformatics Foundation has been awarded a significant hardware grant by Sun Microsystems.

  • Ewan Birney is delivering a keynote address and several open-bio people are considering tutorials and presentation proposals for the O'Reilly Bioinformatics Technology Conference in Feb. 2002.

  • Jason Stajich has issued the Bioperl BOSC 2001 meeting report. The report outlines our plans and directions for the 0.9.x series and the drive to a stable Bioperl 1.0 release.

  • Lincoln Stein has checked Bio::DB::GFF code into the CVS repository. The system is intended to be a lightweight data access layer for sequence annotations. It will support querying both ACeDB and relational databases.

  • Martin Senger is looking for comments and suggestions regarding Novella, a "bridge" project aimed at providing easy access to CORBA BSA services.

[May / June 2001]

  • Bioperl-0.7.1 has been released

  • [ tarball available ] We need people to help test the 0.7.1 release code on as many operating systems as possible. Pending bugs holding up the 0.7.1 release are tracked in this Wiki page.

  • Jason suggests that adding Grail functionality would be a good project for new bioperl people who are interested in contributing to the project. Doug Hyatt, author of GrailEXP responds with an offer to help out.

  • Heikki has committed modules for reading and writing PHYLIP (interleaved only) and NEXUS sequence formats.

  • bioperl.org server has switched to OpenSSH. This message explains why users may see key related error messages.

  • Heikki Lehvaslaiho has joined the core group.

  • Ewan Birney got bored on a train ride and used Lincoln Stein's universal DB idea to implement Bio::DB::Universal. The module is an artifical datbase that attempts to delegate to the proper database (web or local) based on the id.

  • Jason Stajich announces roadmap and todo list for BioPerl 0.7.1 and beyond

  • Lincoln Stein cooks up Bio::DB::Fasta allowing fast indexed access to a directory of Fasta formatted files.

  • Mark Wilkinson has checked the SeqCanvas v2 viewer into bioperl-gui

  • Jason Stajich has committed an initial Dia UML diagram of Bioperl objects & Heikki Lehvaslaiho has added diagrams for Bio::Variation. Look for them in the bioperl-live repository under models/.

  • Ewan Birney uses bioperl-db to implement a caching BioCorba server

    [October - November 2000]

    • Bioperl 0.6.2 has been released. It is mostly a bugfix release with very few functional changes.This should be the final 0.6 series release. The main branch of our current CVS repository (bioperl-live) contains significant new code and functionality that is *NOT* part of the 0.6.2 release.

    • We are working on coordinating a Bioperl 0.7 release that will merge all the new features into the main distribution. We estimate that Bioperl 0.7 will come out by year end. Scanning the bioperl-l archives is a good way to see what new code and features have been added.
