Project: Cookbook
This page describes the outline of a project called the BioPerl Cookbook. It is called after the Perl Cookbook by Tom Christiaensen and Nathan Torkington (a book you should have when you're serious about programming in Perl). Like it ancestor it will contain small, well documented pieces of code (called 'recipes') which will explain the different areas of BioPerl.

After discussion on the developpers mailing list, the following points were agreed upon (this doens't mean that these are set in stone, if you have comments, remarks, adddtions, ... feel free to discuss this on the guts-mailing list or contact me directly krbou@pgsgent.be).

Format

The "book" will be also made available in a 'pretty-printable' format (PostScript, pdf, ...). As I have very limited (nearly no) experience with Tex/Latex or DocBook, there is a chance that the first release might not be in such a format. (if you have experience in writing a technical book or user guide feel free to contace me).

Release Dates The target date for the first release is together with the 0.6 release.

Contents The current proposal for the chapters on the different areas of BioPerl

  • basic sequence manipulation
  • sequence/database managment
  • advanced sequence manipulation/analysis
  • sequence similarity methods
  • basic allignment methods
  • similarity search post-processing
  • other biology-related perl modules
    • Aceperl
    • BoulderIO
    • GFF
    • ...
  • advanced applications (putting it all together)
  • ...

General topics (which will mostly consist of links to relevant documentation)

  • Perl (language info, installation, books, ...)
  • Perl modules (installing, using, ...)
  • BioPerl (general intro, prerequisites, installation, other info, ...)
  • Related programs
  • work in other languages
  • ...

Structure of a chapter

  • Introduction (short, general introduction to the subject of the chapter)
  • Recipes (the different examples), organized with increasing difficulty. Each recipe consists of
    • problem
    • solution (the actual code)
    • discussion (disccussion of the solution with possible alternatives)
    • see also (references to other recipes, documentation, ...)
  • Reference documentation (the docs that come with the modules themself).
