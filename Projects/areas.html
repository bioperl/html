Areas of interest
This list is to idenitify key areas where bioperl might provide sensible solutions for bioinformatics.

For each area we give a number of criteria about its status in bioperl

  • an indicator of importance ranked from 0 to 5, with 0 being the most important
  • a percentage of how much functionality we know of in bioperl that covers this area
  • whether or not this functionality exists in a working form in the current CORE modules (as of Feburary 1999).
AreaDescriptionImportance%doneIn CORE?notes
Sequence data manipulation Translation, truncation and lightweight analysis of sequences 0 70% yes Restriction enzyme and Signal peptide are examples of lightweight analysis
Sequence Similarity Searches BLAST, Fasta, Smith-Waterman, HMMer/SAM, Wise2 1 40% yes Only BLAST is in the core modules, but we think it covers at least 40% of this area
Sequence Database Access Web (NCBI,SRS), Flatfile, RDB, ACeDB 1 10% yes The DB directory due out in 0.05 has this info, but only currently covers Web
Sequence Alignment Protein to Protein, DNA to DNA, DNA to protein etc 1 30% yes We cannot handle complex DNA to protein alignments. Alignment creation is not well covered See Projects/SeqAlign
Sequence Annotation Attaching computational results and other annotations to a single sequence 1 10% no We have Ian Korf's SeqFeature, but no parent object
3D Structures Data Models for biologically relevant 3D structures 2 20% no A start made at bioperl. See Projects/Structure
Sequence/Structure families Collections of related sequences or structures 2 30% no Preliminary work on SCOP and Pfam.
Genomic Data Manipulation Handling whole genome datasets sensibly 2 15% no SacchDB work and also Aceperl can provide starting points for this
Phylogenetic Trees Making and handling phylogenetic trees 2 50%(?) no Georg Fuellen has started this project
Clustering sequences/structures/literature Generic mechanisms for making,handling and traversing clusters 3 10% no This maybe solved better in other fields. GeneFammer from Jong Park may solve this
Comparitive Genome Analysis Combining whole genome analysis from a number of genomes 3 0% no This requires genome data manipulation to be well solved
Biological Literature Fetching and handling biological literature (PubMed,online journals) 4 20%(?) no We are aware of at least one effort in this area. XML may have a big role in this
Analysis Workflow Scheduling, running and versioning analysis on sequences 4 5% no To a certain extent, the current core BLAST modules
Sequence Assemblies Assemblies of traces from shotgun to make a sequence 5 0% no We don't see a big role for bioperl in this, as it typically intergrated into LIMS system and heavyweight GUIs
Proteomics 2D gel analysis, 2-hybrid system and mass spec 5 0% no 2D gels require image analysis, which perl is not well suited to. Other things could be interesting
