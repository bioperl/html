Preliminary requirements list for a Structure object

This is an initial listing. It is not necessarily complete. Not all of the things listed here are expected to go into the Structure object itself. Think of this is a rough set of use-cases for biomolecular structure information, which should be considered when designing a structure object.

Send suggestions to sac@neomorphic.com

The requirements seem to fit into 5 main categories:

  1. Atomic coordinates
  2. Structural features
  3. Analyzing & editing structures
  4. Sequence information
  5. Database information

  1. Atomic Coordinates

    1. Parse and output different structure file formats, at a minimum PDB but could include mmCIF, CML/XML, and other formats.
    2. Be able to obtain different representations the structural coordinates. For example, some applications require a hierarchical representation (complex, molecule, domain, residue, atom) while other might require a non-hierarchically representation (a simple array of atoms or a graph of atoms and bonds). Some applications may utilize minimal representations of the structure, such as only backbone atoms (C-alpha or mainchain form proteins; phosphodiester backbone for nucleic acids) or heavy-atom only (no hydrogens, no waters).
    3. Convert between alternate representations of the structure.
    4. Molecular coverage: at a minimum, support protein and nucleic acid xtallograpic and NMR structures and their heterogens. Multi-chain and multi-model (NMR) structures should also be supported.
    5. Theoretical structural models: Obtain quality factors and other relevant modeling statistics. this will depend on the modeling method. For a homology-based model, methods should be available to get the structure and region of the structure the model is based on.

  2. Structural Features

    1. Provide methods for querying structure for statistics regarding atoms, hetatoms, connectivity, secondary structural elements, disulfide bonds, presence of alternate conformations, structure quality factors (resolution, R factor), crystal packing group, and other data obtainable directly from a PDB file. Specific examples:
      • get the start and end residue numbers for each chain in a structure.
      • get the chain association for a given hetero group.
    2. Obtain all atomic details for a given atom (xyz, atom serial number, atom name, chain name, residue name, residue number, occupancy, B-factor). For example, given an atom, determine what residue and what molecule (chain) it is in.
    3. Obtain information regarding what domains and other structurally or biologically relevant information such as the locations of active sites, metal binding site, covalently modified residues, etc.
    4. Need a robust domain-definition mechanism:
      • how defined (SCOP, CATH, etc.)
      • interchain, intra-chain, contiguous, discontiguous.
      • Many of the requirements of the structure object apply at the level of the individual domain as well.
    5. Need a flexible mechanism for specifying locations of features.

  3. Analyzing & Editing Structures

    4. Structural analyses can be rather compute intensive and so it makes sense to not burden the structure object with complex analytical methods. Such methods are best left for a structural utility module or application. Still, it's useful to consider these analysis from the point of view of what requirements they impose on a structure object.
    1. Display/select acidic, basic, polar and non-polar amino acids, hydrogen bonds
    2. Charge molecule and compute energy, hydrogen bonds, salt bridges.
    3. Given sequence alignment, compute optimal structure superposition (produce a rotation/translation matrix and/or RMSD).
    4. Given a matrix, compute the RMSD.
    5. Add/remove/change residues, hydrogens, and water molecules.
    6. Energy minimize an altered structure.

  4. Sequence Information

    1. Obtain a representation of the structure as sequence or a set of sequences (strings as well as specialized objects).
    2. Get sequence from the actual set of residues in the atomic coordinate set.
    3. Map between residue numbers between the atomic coordinate set and the numbers in the sequence string (say from a Fasta representation).
    4. Specify and generate subsections of the structure given a subsequence (range of integer positions as well as a sequence object).

  5. Database Information

    1. Build a structure object given a valid structural database identifier.
    2. Obtain information about the structure given a valid structural database identifier (molecule name, source organism, deposition date, etc.).
    3. Support local and remote database access. Many groups store versions of the PDB locally or may have their own private structural database.
    4. Get URLs for web-based information about the structure.
