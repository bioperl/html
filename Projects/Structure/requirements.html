Preliminary requirements list for a Structure object

This is an initial listing. It is not necessarily complete. Also, not all of the things listed here need to go into the Structure object itself. Think of this is a rough set of use-cases for biomolecular structure information, which should be considered when designing a structure object.

Send suggestions to sac@neomorphic.com

The requirements seem to fit into 6 main categories:

  1. Atomic coordinates
  2. Structural features
  3. Analyzing structures
  4. Editing structures
  5. Sequence information
  6. Database information

  1. Atomic Coordinates

    1. Parse and output different structure file formats, at a minimum PDB but could include mmCIF, CML/XML, and other formats.
    2. Be able to obtain different representations the structural coordinates. For example, some applications require a hierarchical representation (complex, molecule, residue, atom) while other require a non-hierarchically representation (an array of atoms). Some applications may utilize minimal representations of the structure, such as only backbone atoms (C-alpha or mainchain form proteins; phosphodiester backbone for nucleic acids) or heavy-atom only (no hydrogens, no waters).
    3. Convert between alternate representations of the structure.
    4. Molecular coverage: at a minimum, support protein and nucleic acid xtallograpic and NMR structures and their heterogens. Multi-chain and multi-model (NMR) structures should also be supported.
    5. Theoretical structural models: Obtain quality factors and other relevant modeling statistics. this will depend on the modeling method. For a homology-based model, methods should be available to get the structure and region of the structure the model is based on.

    1. Structural Features

      2. Provide methods for querying structure for statistics regarding atoms, hetatoms, connectivity, secondary structural elements, disulfide bonds, presence of alternate conformations, structure quality factors (resolution, R factor), crystal packing group, and other data obtainable directly from a PDB file.
      3. Obtain all atomic details for a given atom (xyz, atom serial number, atom name, chain name, residue name, residue number, occupancy, B-factor). For example, given an atom, determine what residue and what molecule (chain) it is in.
      4. Obtain information regarding what domains a given structure contains.

      1. Analyzing Structures

        1. Display/select acidic, basic, polar and non-polar amino acids, hydrogen bonds
        2. Charge molecule and compute energy, hydrogen bonds, salt bridges.
        3. Compute the RMSD between to structures. Specify which regions of each structure are to be superimposed.
        4. Compute a rotation/translation matrix for creating a minimum RMSD alignment between structures or structural sub-regions.

        1. Editing Structures

          1. Add/remove/change residues, hydrogens, and water molecules.
          2. Energy minimize an altered structure.

          1. Sequence Information

            2. Obtain a representation of the structure as sequence or a set of sequences (strings as well as specialized objects).
            3. Get sequence from the actual set of residues in the atomic coordinate set.
            4. Map between residue numbers between the atomic coordinate set and the numbers in the sequence string (say from a Fasta representation).
            5. Specify and generate subsections of the structure given a subsequence (range of integer positions as well as a sequence object).

            1. Database Information

              2. Build a structure object given a valid structural database identifier.
              3. Obtain information about the structure given a valid structural database identifier.
              4. Get URLs for web-based information about the structure.
