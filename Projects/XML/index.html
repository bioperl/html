Purpose

The purpose of this web site is to coordinate information dissemination and discussion on the use of XML in bioinformatics, and to serve as a repository for metadata describing the relationships between  biological XML data resources.  My current thinking is that the latter should be accomplished through RDF, but since this is still in draft stage, this thinking may change.

FAQ (perl XML FAQ)

What problems can XML solve?

Biology is a complex discipline and a wide variety of data resources and repositories have been developed to support biological research.  Many of these are interrelated, but it is currently difficult to identify or use these in relationships computationally because the different data sources use incompatible formats and semantics.  XML offers a way to  serve and describe data in a uniform and automatically parseable format.

XML also provides an open source solution for data migration between programming languages.  Using modules such ash XML::Parser, it is possible to move a structured data object between C/C++, java, and PERL without having to worry about the encoding/decoding details.

What problems will not be solved by XML?

XML is basically a language for describing data, but it is not the description.  We need to develop that ourselves.  Just because BSML and BIOML both use XML to encode data does not mean that a BSML browser will be able to view BIOML data.  The fact that both are using XML means that writing a filter to convert one to the other can skip alot of the low level parsing that would otherwise be needed.

Why use XML in biocomputing?

XML is an easily and automatically parseable way to present data on the web.  The basic representation uses standard ASCII text so you can view, edit, and debug data with standard text tools like emacs, vi, grep, and wc. Most importantely, a strong industry consortium is developing that will support the use of XML as the underlying technology for a next generation internet.

What is the relationship of XML to BIO.PERL?

At the moment, BIO.PERL.org is "looking with interest" at XML, but there is no formal relationship.  PERL is a good tool to use with XML (see modules below), and XML hopefully become a standard for biological data resources, so BIO.PERL may, in the future, incorporate XML technology into the BIO.PERL toolkit.

What is the status of XML?

XML itself is a well-defined standard that has been promulgated by the W3C organization.  At present, few tools or data resources making use of this standard are available.  One of the goals of the XML initiative is to facilitate the development of metadata resources (resources describing other resources), but at present this field is in rapid flux and the RDF format itself remains in draft stage.

Is XML supported by the major databases in biology?

At present, none of the major database support an XML interface, although almost all of them are actively exploring the use of XML for future work.

Correctness in XML - well formed & valid

XML data can have varying levels of "correctness".   The first requirement is that the data be "well formed" meaning that tags are syntactically correct and nested tags comply with a basic hierarchical data format.   A more stringent requirement is based on the use defined data type declarations.   XML data that specifies and obeys a data type definition (DTD) is said to be "valid".  An XML parser that looks for and uses a DTD in parsing the data is said to be a "validating parser".  Unfortunately, many of the code samples currently on the net are neither well formed or valid, in part because validating parsers have only recently been widely available.

Software distributions

DTD (data type defintion files)

  • bioml.dtd - a slightly edited version of the Proteomics BIOML data type definition that supports validated parsing of sequence data.  See insulin2.xml for a sample of XML data.

XML perl modules - The CPAN distribution of PERL modules for parsing and manipulating XML.   Includes:

  • XML::CGI - a module to convert CGI parameters to and from XML
  • XML::DOM - a Perl extension to XML::Parser. It adds a new 'Style' to XML::Parser,called 'Dom', that allows XML::Parser to build an Object Oriented data structure with a DOM Level 1 compliant interface.
  • XML::Dumper - a simple package to experiment with converting Perl data structures to XML and converting XML to perl data structures.
  • XML::Encoding - a subclass of XML::Parser, parses encoding map XML files.
  • XML::Generator is an extremely simple module to help in the generation of XML.
  • XML::Grove - provides simple objects for parsed XML documents. The objects may be modified but no checking is performed.
  • XML::Parser - a Perl extension interface to James Clark's XML parser, expat
  • XML::QL - an early implementation of a note published by the W3C called "XML-QL: A Query Language for XML".
  • XML::XQL - a Perl extension that allows you to perform XQL queries on XML object trees.

Non-CPAN sites

Bioinformatics

BSML - Bioinformatic Sequence Markup Language - a proposed data type definition for the representation of molecular biological data in XML developed by Visual Genomics and used as the basis for the BSML web browser developed and distributed by the same company.

BIOML - Biopolymer Markup Language - a proposed data type definition for molecular biological sequence and structure data in XML developed by ProteoMetrics, LLC.

BIO.PERL.ORG - a site dedicated to the development of PERL tools for molecular biology computing, including multiple discussion forums, web archives, and pointers to software distributions
