Project: Sequence Annotation
There are two main objects involved in annotations. Sequence Features (defined as an interface in Bio::SeqFeatureI) and Annotations (Bio::Annotation, not defined as an interface yet). Sequence features have a location on the sequence, whereas Annotations are textual annotations usually associated with a particular sequence and not a region.

These objects were primarily designed to allow reading and writing of EMBL and GenBank sequence entries, without tying them religiously to these formats.

Bio::SeqFeatureI inheriets from Bio::RangeI which defines that Ranges have to have a start, an end and a strand, and also provides a number of useful helper functions, such as $range->overlaps($otherange) which tests for overlap.

Bio::SeqFeatureI also defines information to be feature table and GFF compatible. This ensures this object is well mapped to these flat file formats.

Annotation objects provide references, comments and database link objects, corresponding the similar lines in EMBL/GenBank flat files.
