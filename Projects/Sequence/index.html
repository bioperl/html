Project: Sequence
Sequence objects in 0.6 release have been reimplemented from the previous 0.5 and 0.4 series. There are two main types of objects, and, with the new architecture rules, these are defined both as interfaces ("I" files) and implementations.

The two objects are:

  • PrimarySeq - the sequence and names, which has the functionality of retrieving parts of a sequence and providing both reverse complement and translations for DNA sequence.
  • Seq - sequences with sequence features and annotation. As well as providing access to sequence features, the SeqI interface inheriets from the PrimarySeqI interface, meaning that one can use a Seq object anywhere one can use a PrimarySeq object.
The split between PrimarySeq and Seq seems at first glance quite confusing. Generally most people will use Seq objects nearly all the time. The split is far more useful for using Sequences in larger scale projects. Firstly it means that people can replace the PrimarySeq object with their own implementation without having to worry about sequence features. Secondly it provides a far easier memory management system, without any cycles.

For more information about sequence features and annotations please read sequence annotation
