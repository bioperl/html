Project: Sequence
This page describes the design of an object for a biological sequence and lightweight analyses of sequence data. The existing Bio::Seq.pm module encapsulates data for a single DNA, RNA, or AminoAcid sequence. It also handles some of the lightweight sequence analysis tasks. A lightweight analysis depends primarily on the sequence information per se and does not rely overly on external resources. The analyses included in Bio::Seq are:
  • truncation
  • reverse-complement
  • translation
Parsing and output (I/O) is currently handled in the Bio::Seq.pm object (as of the 0.04 release). ReadSeq (a C sequence parser written by Don Gilbert) was also used to provide an interface to many formats, but this required ReadSeq to be installed and new formats to be intergrated into ReadSeq. Both these facts meant that some parsing code moved into Perl in the Bio::Seq object, which consequently started to fill up with parsing code. Considering the number of formats at large in bioinformatics, a better design is to separate the parsing code from the sequence object.

In the 0.05 releases a new mechanism for handling sequence I/O has been established, (SeqIO). Support for each format is handled by dedicated modules and only compiled if needed. This provides for better encapsulation of the format, a less monolithic Bio::Seq.pm and better extensibility. Eventually all the parsing code should be removed from Bio::Seq.pm once people get used to using SeqIO.

Current formats supported in Bio::Seq directly (old-style)

  • Raw
  • GenBank
  • Fasta
  • GCG
Current formats supported in Bio::SeqIO
  • Fasta
  • Raw
  • EMBL
Bio::Seq future

Although more of a implementation issue, we would like to draw attention to the fact that Bio::Seq has become a rather large and rehacked module. It is the central module for bioperl, and so unsurprisingly the first written, and now the most difficult to maintain.

We think it is sensible to try to move towards reimplementing Bio::Seq in a clean rewrite from well defined requirements. To do this we think the following steps are required

  • A careful description of the requirements of Bio::Seq
  • Development of a comprehensive test suite of Bio::Seq functionality that we wish to preserve
  • A separation of I/O functionality to SeqIO
  • A careful reimplementation of Bio::Seq
Other lightweight analyses provided outside of Bio::Seq

To keep the sequence object simple and maintainable, these sorts of analyses should not be provided by the Bio::Seq object itself. Current modules that provide such lightweight analyses are located in Bio::Tools and are:

  • Restriction analysis (Bio::Tools::RestrictionEnzyme.pm)
  • Signal sequence identification (Bio::Tools::Sigcleave.pm)

Desirable lightweight analyses not yet written

  • Property analysis (MW, pI, etc.)
  • Metric "sliding window" properties (GC-content, hydrophobicity, etc.)
  • Codon usage
  • Sequence/PCR primers (Tm, primer design, etc.)
  • Translation using alternate genetic codes
  • Reverse translation

These lightweight analyses are small, self-contained projects that are in need of eager developers. We invite anyone interested in working on any of these items to post a message to the bioperl discussion list.
