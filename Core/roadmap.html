Bioperl Development Roadmap


Where we are. What we have. Where we are going.

The bioperl sequence object has been integrated into a general framework developed by Steve Chervitz. The current framework has since expanded and is comprised of a set of integrated modules The core distribution is now available worldwide via CPAN. Our modules can be found in the Bio/ directory.

Design
The overall design of the core modules is (roughly) coordinated from the Projects overview page. This page focuses on the Perl module code which is already in the CVS system or expected to be eventually.

Current Efforts
Bioperl is now running smoothly from a central source code repository. The 0.04* release is out and is only being bug-fixed while people add features and work on code for the eventual 0.05 release.

We are busy preparing for the first Bioperl Workshop which will be held right before ISMB-99 in Heidelberg Germany.

Code status & Committments
PreSeq.pm has been replaced by Seq.pm and will become our general flexible and lightweight sequence object. Parse.pm is no longer being used by core bioperl code. We will also be moving all sequence formatting and parsing code out of the sequence object and transitioning over to the SeqIO/* modules

In Progress / Things that may end up in the next release

If you are really curious about how these projects are going, join the guts-list or check the most recent source code dump.
  • Bio::Tools::Fasta - Similar to our robust support for launching and manipulating BLAST search results, we hope to do the same with Fasta.
  • Bio::DB::Genbank - Given an accession number, fetches sequence from NCBI website and returns a bioperl sequence object. Genbank is the first of many databases we hope to make our objects 'aware' of in this way. Aaron Mackey is working on this.
  • Bio::SeqIO - new stream based methods for reading and writing to/from sequence database files. Lots of code coming in-- so far we have SeqIO::Genbank, SeqIO::EMBL, SeqIO::Fasta, SeqIO::Raw, SeqIO::GCG and more on the way.
  • Bio::Tools::Sigcleave - perl emulator of the EGCG Sigcleave utility. Chris Dagdigian is working on this.
  • Bio::SeqFeature and Bio::Entry - The Bio::Entry is the 'heavy weight' sequence object which is more like a representation of a Genbank or EMBL flat file complete with feature table. Ian Korf is working on these objects.
  • Bio::Structure - A number of people have been talking about a Structure object (probably an object heirarchy). More info in the /Projects/ section.
  • Bio::SearchDist - A perl wrapper for Sean Eddy's Extreme Value and Gaussian fitting code (already provided in the bp_sw XS extension). Ewan Birney is working on this.
Steve Chervits, the bioperl coordinator is maintaining and improving the Bio::Tools::Blast and Bio::Root code.
