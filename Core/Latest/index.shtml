Bioperl is a collection of "standard" CPAN type modules so installation should be fairly easy and straightforward.

The latest distribution, bioperl-06 should still be considered a for-unix-only alpha release. We use the package for real work at our sites, but if you are considering using bioperl in production work, we would recommend both heavily testing the package on your site and posting a mail to discussion list to hear whether anyone else has had problems with the functionality you want to use.

Bioperl-0.6.2 is essentially a bugfix release to our 0.6 distribution. A significant amount of new functionality has been witheld and will appear in the bioperl-0.7 release which we expect will be available before the end of the year.

We are particularly interested in hearing about how the installation process works on different platforms and operating systems. General comments, suggestions and installation war stories should be sent to the developers email-list bioperl-guts-l. Bug reports should be submitted via our web-based reporting system at http://bioperl.org/bioperl-bugs/ or by sending email to bioperl-bugs@bioperl.org.

After obtaining the distribution file, unpack it and read the installation instructions in README file.

The distribution comes with its own internal documentation in the perl documentation format (pod). This documentation is has been translated into HTML links available below. The best starting point for people learning about bioperl is the bioperl.pod documentation, followed by the Bio::Seq.pm documentation. View the distribution files directly:
