About the bioperl project
The Bioperl Project is an international association of developers of open source Perl tools for bioinformatics, genomics and life science research.

We work closely with our friends and colleagues at biojava.org, biopython.org, bioxml.org and biocorba.org. .

The Bioperl server provides an online resource for modules, scripts, and web links for developers of Perl-based software for life science research. We can also provide web, FTP and CVS space for individuals and organizations wishing to distribute or otherwise make freely available standalone scripts & code.

News

[November 2001]

  • bioperl-0.7.2 released.

  • bioperl-db released. bioperl-db provides code and modules for storing and retrieving biological sequence information from relational databases (current version only supports mysql).

  • "Genquire" - A graphical genomic editor/browser built on top of the bioperl and bioperl-gui codebases has been released by Mark Wilkinson and David Block. The Genquire project is hosted at Bioinformatics.org.

  • Kris Boulez commits inital Bio::Structure objects into bioperl-live

  • Bristol-Myers Squibb contributes code for converting bioseq objects to and from the BSML XML format.

  • Have you read the latest Open Bioinformatics Foundation newsletter?

  • Bioperl class diagrams are available in PDF form.

  • (experimental feature) - New PDoc-style documentation for all of the bioperl codebases is online at http://doc.bioperl.org/. It looks to be far more useful than our traditional pod2html translated documentation pages. Please evaluate and send comments to the bioperl-guts-l list.

Script Central
We are looking to archive, mirror or provide pointers to any and all biology-related perl code that is freely available for download. If you have developed a utility or module that you think may be useful to others, please consider placing it in the public domain on your site or ours. Help advance our field!

Jason Stajich has created a scripts repository and is seeking submissions. The goal is to have more utility and example scripts in our upcoming releases.
